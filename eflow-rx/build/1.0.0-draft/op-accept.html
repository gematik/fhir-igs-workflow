Operation $accept (Task akzeptieren)

Seiteninhalt:

  • Nachricht
  • Anforderungen an Schnittstelle
  • API Beschreibung
  • Hinweis
  • Geschäftslogik
  • Sicherheitsanforderungen

Diese Operation basiert auf der gleichnamigen Operation in der Core-Spezifikation.

Nachricht

Die Nachricht wird als HTTP POST an /Task/{id}/$accept gesendet.

Anforderungen an Schnittstelle

  • Ergänzende Server-Anforderungen zu $accept: Anforderungen an den E-Rezept-Fachdienst zur Bereitstellung der Schnittstelle.
  • Ergänzende Client-Anforderungen zu $accept: Anforderungen an den Client des E-Rezept-Fachdienstes zur Nutzung der Schnittstelle.

API Beschreibung

Die API-Beschreibung für den Aufruf der Operation findet sich auf:

  • API-ERP: E-Rezept abrufen

Hinweis

  • Die Rückgabe kann bei Mehrfachverordnung mit 403 scheitern, wenn die Einlösefrist noch nicht begonnen hat.
  • AccessCode und insbesondere secret sind als hochsensible Daten zu behandeln.
